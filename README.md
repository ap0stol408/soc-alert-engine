# 🛡️ SOC Alert Engine

Event-driven SOC alert engine that processes monitoring events from
multiple sources and sends real-time alerts via **WhatsApp and phone
calls**.

This project automates incident notifications for Security Operations
Centers (SOC), helping reduce response time and improve visibility of
critical monitoring events.

------------------------------------------------------------------------

# 🚀 Overview

The **SOC Alert Engine** collects monitoring events from multiple
sources, processes them, and sends notifications through communication
channels such as **WhatsApp and phone calls**.

The engine is designed with a modular architecture so new event sources
and notification channels can be added easily.

Typical workflow:

    Monitoring Systems (Nagios / Splunk Email Alerts)
                    │
                    ▼
            SOC Alert Engine
                    │
                    ▼
     Notification Channels (WhatsApp / Phone Calls)

This allows SOC analysts to receive **instant alerts on mobile devices**
when critical events occur.

# 🧠 Architecture

The project follows a modular design that separates **event ingestion,
processing, and notification delivery**.

    src/
    │
    ├── core/
    │   ├── alertEngine.js
    │   └── stateManager.js
    │
    ├── pollers/
    │   ├── nagiosPoller.js
    │   └── emailPoller.js
    │
    ├── parsers/
    │   └── emailParser.js
    │
    ├── adapters/
    │   └── twilioClient.js
    │
    └── index.js

### Components

**Pollers**

Collect monitoring events from external systems.

-   Nagios polling
-   Email monitoring via IMAP

**Parsers**

Extract structured alert data from incoming messages.

**Core Engine**

Responsible for alert processing and state management.

-   Processes alert events
-   Manages alert state
-   Prevents repeated processing

**Adapters**

Send alerts through external services.

-   Twilio integration for WhatsApp notifications

# 🔧 Installation

Clone the repository:

``` bash
git clone https://github.com/ap0stol408/soc-alert-engine.git
cd soc-alert-engine
```

Install dependencies:

``` bash
npm install
```

------------------------------------------------------------------------

# ⚙️ Configuration

Create a `.env` file based on the following example:

``` env
# Monitoring Nodes
MONITOR_NODE_1=monitor-node-ip
MONITOR_NODE_1_STATUS_PATH=/path/to/nagios/status.dat

MONITOR_NODE_2=monitor-node-ip
MONITOR_NODE_2_STATUS_PATH=/path/to/nagios/status.dat

SSH_USER=monitor_user
POLL_INTERVAL=60


# Email Alert Source (IMAP)
IMAP_HOST=mail.example.com
IMAP_PORT=993
IMAP_SECURE=true
IMAP_USER=alerts@example.com
IMAP_PASS=your_email_password


# Twilio Configuration
TWILIO_ACCOUNT_SID=your_account_sid
TWILIO_AUTH_TOKEN=your_auth_token

TWILIO_WHATSAPP_FROM=whatsapp:+1415XXXXXXXX
TWILIO_WHATSAPP_TO=whatsapp:+521XXXXXXXXXX
```

------------------------------------------------------------------------

# ▶️ Running the Engine

Start the alert engine:

``` bash
npm start
```

The engine will begin polling monitoring sources and sending alerts when
events are detected.
